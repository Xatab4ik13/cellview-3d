
## Распределение прибыли (Revenue Recognition)

### Проблема
Клиент платит 14 400₽ за год одним платежом → вся сумма падает в выручку марта. Нужно распределять по месяцам.

### Решение

**1. Новая таблица `revenue_entries`** (миграция на сервере):
```sql
CREATE TABLE revenue_entries (
  id VARCHAR(36) PRIMARY KEY,
  rental_id VARCHAR(36) NOT NULL,
  customer_id VARCHAR(36) NOT NULL,
  cell_id VARCHAR(36) NOT NULL,
  month DATE NOT NULL,          -- первое число месяца (2026-03-01, 2026-04-01, ...)
  amount INT NOT NULL,           -- сумма за этот месяц
  payment_id VARCHAR(36),        -- связь с платежом
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**2. Бэкенд — автозаполнение при создании аренды:**
При создании аренды (POST /api/rentals) или регистрации платежа — автоматически создавать записи в `revenue_entries`, разбивая `totalAmount` на помесячные доли.

Пример: аренда 12 мес, первые 6 по 1100₽, вторые 6 по 1300₽ → 12 записей.

**3. API — новый эндпоинт `/api/revenue`:**
- GET /api/revenue?from=2026-01&to=2026-12 — помесячная выручка
- Используется дашбордом вместо текущего подсчёта по дате платежа

**4. Дашборд CRM — график «Выручка по месяцам»:**
Переключить с подсчёта по `paidAt` на данные из `revenue_entries`.

### Что НЕ меняется
- Раздел «Платежи» — по-прежнему показывает факт оплаты (когда деньги получены)
- Существующие аренды — можно заполнить задним числом через скрипт
